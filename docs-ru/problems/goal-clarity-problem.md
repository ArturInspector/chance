# Проблема неясности целей (Goal Clarity Problem)

## Формулировка

**Основное утверждение:**

Большинство целей формулируются нечётко. Неясность цели делает невозможным построение корректного плана и измерение прогресса.

```
∀ G (цель): 
  if not clear(G) 
  then impossible(plan(G)) ∧ impossible(measure(G))
```

## Типы неясности

### 1. Отсутствие критерия завершения

```
unclear_completion(G) ⟺ ∄ criterion(G): done(G)
```

**Примеры:**

| Цель | Проблема |
|------|----------|
| "Выучить Python" | Когда считается выученным? |
| "Стать лучше" | В чём измеряется "лучше"? |
| "Заниматься спортом" | Сколько? Как часто? Как долго? |

### 2. Неизмеримость результата

```
unmeasurable(G) ⟺ ∄ function f: measure(G) → ℝ
```

**Примеры:**

| Цель | Почему не измерима |
|------|-------------------|
| "Быть счастливым" | Нет объективной метрики счастья |
| "Улучшить здоровье" | Слишком общо, много параметров |
| "Развиваться" | Не определено направление |

### 3. Смешение средства и цели

```
confusion(G) ⟺ G описывает процесс, а не результат
```

**Примеры:**

"Читать книги" — это средство или цель?
- Если цель — знание, то нужно измерять усвоенное, а не прочитанное
- Если цель — сам процесс чтения, то зачем?

### 4. Зависимость от внешних агентов

```
uncontrollable(G) ⟺ achieve(G) требует действий других агентов
```

**Примеры:**

| Цель | Зависимость |
|------|-------------|
| "Получить работу в Google" | Решение Google |
| "Найти партнёра" | Решение другого человека |
| "Стать популярным" | Решение аудитории |

## Математическая формализация неясности

### 1. Мера ясности

```
clarity(G) = measurable(G) × controllable(G) × atomic(G)
```

где каждый компонент ∈ [0, 1].

**Полностью ясная цель:**

```
clarity(G) = 1 ⟺ 
  (1) ∃ objective criterion: done(G)
  (2) ∃ procedure P: controllable(P) ∧ execute(P) ⟹ achieve(G)
  (3) G не требует дальнейшей декомпозиции для понимания
```

**Полностью неясная:**

```
clarity(G) = 0 ⟺ нет критерия ∨ нет контроля ∨ требует декомпозиции
```

### 2. Градиент ясности

```
clarity_gradient(G) = ∂clarity(G) / ∂refinement
```

Если градиент > 0, цель можно уточнить.
Если градиент = 0, цель уже максимально ясна или не поддаётся уточнению.

### 3. Порог планируемости

```
plannable(G) ⟺ clarity(G) > threshold (обычно 0.7)
```

Ниже порога — невозможно построить корректный план.

## Алгоритм прояснения цели

```
function clarify(G):
  // Шаг 1: Проверка измеримости
  if not measurable(G):
    G = add_criterion(G)
    Prompt: "Как проверить, что цель достигнута?"
  
  // Шаг 2: Проверка контролируемости
  if not controllable(G):
    G = extract_controllable_part(G)
    Prompt: "Что из этого ты можешь контролировать сам?"
  
  // Шаг 3: Проверка атомарности
  if not atomic(G) and not decomposable(G):
    return error("Цель слишком абстрактна, декомпозируй")
  
  // Шаг 4: Проверка ограниченности
  if not finite(G):
    G = set_time_bound(G)
    Prompt: "Какой срок для достижения?"
  
  return G
```

## Примеры трансформации

### Пример 1: "Выучить Python"

```
Initial: "Выучить Python"
  clarity = 0.1 (неизмеримо, нет критерия)

Step 1: Зачем?
  → Для работы программистом

Step 2: Что конкретно нужно для работы?
  → Уметь решать типовые задачи

Step 3: Какие задачи?
  → CRUD, REST API, работа с БД

Step 4: Критерий?
  → Реализовать 3 проекта: ToDo API, Blog API, E-commerce API

Final: "Реализовать 3 REST API проекта с CRUD и БД за 2 месяца"
  clarity = 0.9 (измеримо, контролируемо, конечно)
```

### Пример 2: "Быть здоровым"

```
Initial: "Быть здоровым"
  clarity = 0.0 (неизмеримо, нет критерия)

Step 1: Что значит здоровым?
  → Хорошие анализы, нормальный вес, физическая форма

Step 2: Какой параметр выбрать?
  → Начну с веса (измеримо)

Step 3: Какая цель по весу?
  → Снизить с 95 до 85 кг

Step 4: За какой срок?
  → 5 месяцев (2 кг/месяц безопасно)

Step 5: Как контролировать?
  → Дефицит калорий + тренировки

Final: "Снизить вес с 95 до 85 кг за 5 месяцев через дефицит 300 ккал/день + 3 тренировки/неделю"
  clarity = 0.95 (измеримо, контролируемо, конечно)
```

### Пример 3: "Получить работу в Google"

```
Initial: "Получить работу в Google"
  clarity = 0.2 (не контролируемо)

Step 1: Что ты контролируешь?
  → Подготовку к собеседованию, отправку резюме

Step 2: Извлечь контролируемую часть
  → "Подготовиться к собеседованиям в топ компании"

Step 3: Что значит подготовиться?
  → Решить 150 задач LeetCode, изучить system design

Step 4: Критерий готовности?
  → Успешно пройти 3 mock interview

Final: "Решить 150 задач LeetCode + пройти 3 успешных mock interview за 3 месяца"
  clarity = 0.85 (измеримо, контролируемо)
```

## Проблема эволюции целей

### 1. Цели меняются в процессе

```
G(t) ≠ G(t + Δt)
```

Что казалось важным в начале, может потерять смысл.

**Пример:**

```
t=0: "Стать Senior разработчиком"
t=6 месяцев: "Работать над интересными проектами" (осознал, что титул не важен)
```

### 2. Разрешение неопределённости

Неясные цели могут быть **временно полезны** для exploration:

```
explore(G_unclear) → discover(G_clear)
```

**Стратегия:**

```
if clarity(G) < 0.3:
  // Слишком неясно для планирования
  // Но можно исследовать направление
  execute(small_experiment(G))
  G' = refine(G, результат эксперимента)
```

### 3. Баланс ясности и гибкости

Слишком жёсткая цель → потеря адаптивности

```
optimal_clarity ∈ [0.7, 0.9]
```

Достаточно ясна для планирования, но гибка для корректировки.

## Практические следствия

### 1. Процедура уточнения обязательна

```
∀ G: before(plan(G)) → execute(clarify(G))
```

Без уточнения план будет неполным (см. Plan Correctness Problem).

### 2. Итеративное уточнение

```
G₀ → clarify → G₁ → execute(small_step) → refine → G₂ → ...
```

Не пытаться прояснить идеально с первого раза.

### 3. Разделение по горизонту

```
Short-term goal (1-3 месяца): clarity → 0.9
Mid-term goal (6-12 месяцев): clarity → 0.7
Long-term goal (1+ год): clarity → 0.5
```

Чем дальше горизонт, тем ниже допустимая ясность.

## Вывод

**Неясность целей — основной источник провалов.**

Система «Шанс» требует:
1. Явное прояснение каждой цели перед планированием
2. Извлечение контролируемой части из неконтролируемых целей
3. Добавление измеримых критериев
4. Итеративное уточнение в процессе выполнения

```
clear_goal = foundation(successful_execution)
```

Без ясности цели нет плана. Без плана нет прогресса.
