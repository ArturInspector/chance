# Аксиомы проекта «Шанс»

Аксиомы — базовые утверждения, которые мы принимаем без доказательств. Вся остальная система строится на них. Если хоть одна из аксиом ложна, вся система рушится.

## 1. Агентность

Человек может совершать действия, изменяющие реальность.
```

Если ты не можешь ничего сделать (полный детерминизм), эта система не для тебя. Но раз ты читаешь и думаешь об этом — ты агент. Действия имеют последствия.

## 2. Измеримость

```
Любое действие вызывает наблюдаемый feedback.
```

Нет действий без последствий. Feedback может быть отложенным (через месяцы), зашумлённым (много факторов), но он есть. Без измерения нет управления.

## 3. Декомпозируемость

```
Любое сложное действие раскладывается на атомарные шаги.
```

Нет магии. «Стать программистом» = тысяча конкретных действий (написать функцию, решить задачу, прочитать главу). Всё можно разложить до «что именно делаешь руками».

## 4. Ограниченность ресурсов

```
Время, энергия, внимание, деньги — всегда конечны.
```

Нельзя делать всё. Каждое действие имеет цену. Каждый выбор исключает другие варианты. Приоритизация — не опция, а необходимость.

## 5. Субъективность наблюдения

```
Объективная реальность недоступна. Ты видишь свою интерпретацию.
```

«Я написал функцию» — это твоя интерпретация. Может, код плох. Может, не то решает. Полная объективность невозможна. Калибровка через внешний feedback — единственный путь к приближению.

## 6. Непроверяемость плана

```
Корректность плана можно узнать только после выполнения.
```

План — гипотеза, не гарантия. Скрытые условия вылезают только на практике. «Делаю всё правильно, но масса не растёт» → где-то ошибка в предпосылках (сон, стресс, генетика). Итеративная коррекция обязательна.

## 7. Гомеостаз

```
Система (мозг, тело) сопротивляется изменениям.
```

Новое дорого. Adaptation cost >> base cost. Мозг не даст делать 10 новых процедур сразу — сожрёшь весь энергобюджет. Привыкание снижает сопротивление, но требует времени (~21 день).

## 8. Неполнота самопознания

```
Ты не можешь обнаружить все свои ошибки изнутри.
```

Паразитные цели невидимы для тебя. Когнитивные искажения — слепое пятно. Теорема Гёделя для жизни: система с ошибками не видит все свои ошибки. Внешние референсы (люди, формальные критерии, данные) обязательны.

## 9. Отложенный feedback

```
Результат приходит с задержкой. Иногда большой.
```

Код — минуты. Продукт — недели. Здоровье — месяцы. Репутация — годы. Чем длиннее feedback loop, тем выше риск. Leading indicators (опережающие метрики) критичны для долгосрочных целей.

## 10. Неопределённость будущего

```
Будущее непредсказуемо. Даже при идеальном плане.
```

Мир хаотичен. Внешние факторы неконтролируемы. Чёрные лебеди случаются. Гибкость и способность адаптироваться важнее точности начального плана.
